STM32 心率血氧体温监测系统 - 虚假数据问题修复
================================================================================
日期: 2026-05-17
问题: 没有手指时仍然输出虚假数据（HR:75, SpO2:98）
解决方案: 添加双通道信号有效性检测机制

================================================================================
核心改动: 人体检测系统调试完成版\HARDWARE\MAX30102\max30102.c
================================================================================

新增函数: max30102_is_finger_detected()
───────────────────────────────────────
用途：判断是否检测到有效的手指接触信号

检查内容：
1. RED和IR信号强度 (均需 > 5000)
2. RED和IR信号饱和度 (均需 < 250000)  
3. RED/IR比值 (需在 0.4-3.0 范围内)

修改的函数：
───────────────────────────────────────
get_real_heart_rate()
  - 无有效信号时：返回 0 (之前会返回虚假值)
  
get_real_spo2()
  - 无有效信号时：返回 0 (之前会返回虚假值)

================================================================================
参数调整指南
================================================================================

如果实际测试中需要调整灵敏度，修改以下参数：

信号强度阈值（max30102.c 第236行）：
  if(red < 5000 || ir < 5000) return 0;
  └─> 可调整至 2000-3000 提高灵敏度
  └─> 可调整至 8000-10000 提高容限度

信号比值范围（max30102.c 第242行）：
  if(ratio < 0.4f || ratio > 3.0f) return 0;
  └─> 可适当放宽范围 (如 0.3-3.5) 提高容限
  └─> 可适当收紧范围 (如 0.5-2.5) 提高准确性

================================================================================
故障排除
================================================================================

现象: 有手指时心率仍显示 0
解决: 
  1. 检查 MAX30102 硬件连接
  2. 将阈值降低 (如改为 2000) 重新编译测试
  3. 确认传感器和手指接触良好

现象: 无手指时仍有虚假值
解决:
  1. 提高阈值 (如改为 8000)
  2. 调整 RED/IR 比值范围
  3. 检查环境干扰（光线、温度）
